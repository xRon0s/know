ハードニング

1. 概要

ユーザー管理強化、SSH公開鍵認証、UFW、Fail2banの要素を組み合わせることでサーバーを強固なものに構築します。Fail2ban以外は権限付与等の手順で設定したので、Fail2banの設定を行います。

2. 設定

以下のコマンドを用いてFail2banをインストールします。

terminal
sudo apt install fail2ban -y # Fail2banをインストール

設定ファイルを直接編集するのではなく、コピーしてカスタマイズ用のファイルを作成します。

terminal
sudo cp /etc/fail2ban/jail.conf /etc/fail2ban/jail.local

jail.localファイル内で、SSHに関する設定をカスタマイズします。[sshd]セクションを見つけて、以下のように変更・追記します。

/etc/fail2ban/jail.local
# --- 設定内容 --- [DEFAULT] bantime = 1h # BANする期間を1時間に設定 findtime = 10m # 試行を検出する期間を10分に設定 maxretry = 5 # 期間内の最大試行回数を5回に設定 [sshd] enabled = true # SSHDに対する監視を有効化 port = 2222 # 監視するSSHのポート番号 logpath = /var/log/auth.log # 監視するログファイルのパス backend = systemd # ログの監視方法 # --- ファイル情報 --- # 所有者: root:root # 権限: 644 (-rw-r--r--)

設定の変更が完了したら、以下のコマンドで設定を適用し状態を確認します。

terminal
sudo systemctl restart fail2ban # Fail2banサービスを再起動して設定を反映させる

3. 確認

Webサーバー上で内容が表示されるか確認します。今回はNginxを用いて確認します。以下のコマンドでNginxをインストールします。

terminal
sudo apt install nginx -y # Nginxをインストール

インストールが完了したら、ブラウザで http://[ラズパイのIP] にアクセスし、"Welcome to nginx" の表記がでるか確認します。
